Return profiling information for the command associated with event.

Parameters:
name specifies the profiling data to query.
param is a pointer to memory where the appropriate result being queried is returned. If param is NULL, it is ignored.

The unsigned 64-bit values returned can be used to measure the time in nano-seconds consumed by OpenCL commands. OpenCL devices are required to correctly track time across changes in frequency and p-states. The CL_DEVICE_PROFILING_TIMER_RESOLUTION specifies the resolution of the timer i.e. the number of nanoseconds elapsed before the timer is incremented.

Returns:
One of the following values:
  • CL_SUCCESS if the function is executed successfully and the profiling information has been recorded
  • CL_PROFILING_INFO_NOT_AVAILABLE if the profiling information is currently not available (because the command identified by event has not completed)
  • CL_INVALID_VALUE if param_name is not valid.
Note:
In the case that exceptions are enabled and error value other than CL_SUCCESS is generated, then cl::Error exception is generated.
